Architektura Asynchroniczna

Analiza kodu FlightScanner.Client

1. Zarządzanie czasem i anulowaniem

Bezpieczeństwo
Zastosowanie mechanizmu CancellationTokenSource z limitem czasowym. Jest to kluczowy element zapobiegający "wiszeniu" aplikacji, gdy zewnętrzne API nie odpowiada.
public const int TimeoutMs = 3000;
// ...
using var cts = new CancellationTokenSource(TimeoutMs);
// Przekazanie tokenu dalej:
await httpClient.GetAsync(provider.Endpoint, cts.Token);
Zastosowanie: Gwarantuje, że cała operacja (niezależnie od liczby zapytań) zakończy się (sukcesem lub błędem) w ciągu 3 sekund. Token jest propagowany w dół stosu wywołań.

2. Wzorzec Scatter-Gather (Rozprosz-Zbierz)

Wydajność
Zamiast używać await w pętli (co dałoby sekwencyjne wykonywanie: A...potem B...potem C), kod tworzy listę "gorących" zadań (Tasks), które startują natychmiast, a potem czeka na wszystkie naraz.
var tasks = new List<Task<ProviderResponseDto?>>();

foreach (var provider in providers)
{
    // WAŻNE: Dodajemy zadanie do listy, NIE czekamy (brak await tutaj)
    tasks.Add(GetFlightsFromProviderAsync(...));
}

// Czekamy, aż najwolniejsze zapytanie się skończy
var results = await Task.WhenAll(tasks);
Zastosowanie: Maksymalizacja wykorzystania łącza sieciowego. Czas wykonania = czas najwolniejszego zapytania, a nie suma wszystkich czasów.

3. Raportowanie postępu (IProgress<T>)

Dobre Praktyki
Abstrakcja pozwalająca metodzie asynchronicznej informować "świat zewnętrzny" o postępach bez wiedzy o tym, czy jest to aplikacja konsolowa, czy okienkowa (WPF/WinForms).
IProgress<string> progress = new Progress<string>(Console.WriteLine);

// Wewnątrz metody asynchronicznej:
progress.Report($"[Phase 1] Found {providers.Count} providers.");
Zastosowanie: Separacja logiki biznesowej od UI. W aplikacjach UI Progress<T> automatycznie dba o SynchronizationContext (powrót na wątek główny), w konsoli działa bezpośrednio.

4. Izolacja błędów w Taskach

Stabilność
Metoda GetFlightsFromProviderAsync posiada własny blok try-catch i w razie błędu zwraca null zamiast rzucać wyjątek wyżej.
try { /* logic */ }
catch (Exception ex)
{
    progress.Report($"Error: {ex.Message}");
    return null; // Bezpieczny powrót
}
Zastosowanie: Kluczowe przy Task.WhenAll. Gdyby jeden Task rzucił wyjątek bez tego zabezpieczenia, cała agregacja mogłaby zostać przerwana. Tutaj błąd jednego dostawcy nie ubija całej aplikacji.

5. Async Main (Top-level entry point)

C# Feature
Możliwość oznaczenia metody startowej jako asynchronicznej. Pozwala to na użycie await już na samym początku programu.
public static async Task Main(string[] args)
{
    await RunFlightScannerAsync();
}
Zastosowanie: Eliminuje stare obejścia typu .GetAwaiter().GetResult() lub .Wait(), które mogły prowadzić do deadlocków lub trudnych w analizie wyjątków (AggregateException).

6. Nieblokujące operacje I/O (HttpClient)

Resource Management
Użycie metod kończących się na Async (np. GetAsync, ReadFromJsonAsync).
// Wątek jest zwalniany na czas oczekiwania na odpowiedź HTTP
var response = await httpClient.GetAsync(provider.Endpoint, ct);
Zastosowanie: Podczas gdy kod czeka na odpowiedź sieciową (co trwa np. 200ms), wątek procesora nie jest zablokowany ("busy waiting") i może wrócić do puli wątków, by obsłużyć inne zadania. To podstawa skalowalności.
